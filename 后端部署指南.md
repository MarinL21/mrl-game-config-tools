# 🚀 后端服务部署指南

## 📋 项目说明

这是一个基于 **FastAPI** 的翻译服务后端，使用 **Google Gemini AI** 提供高质量的多语言翻译。

### 优势
- ✅ **安全**：API Key 存储在服务器端，不会暴露给用户
- ✅ **稳定**：统一管理 API 调用，避免客户端直接调用
- ✅ **可控**：可以添加使用限制、日志记录、缓存等功能
- ✅ **免费部署**：支持 Vercel、Railway 等免费平台

---

## 🛠️ 本地开发

### 1. 环境要求

- Python 3.9 或更高版本
- pip（Python 包管理器）

### 2. 安装依赖

```bash
# 创建虚拟环境（推荐）
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 3. 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件，填入您的 Gemini API Key
# GEMINI_API_KEY=AIzaSy...
```

**获取 API Key：** https://aistudio.google.com/app/apikey

### 4. 运行服务

```bash
python main.py
```

服务将在 `http://localhost:8000` 启动

### 5. 测试 API

访问以下地址：

- **API 文档**：http://localhost:8000/docs
- **健康检查**：http://localhost:8000/health
- **支持的语言**：http://localhost:8000/api/languages

### 6. 测试翻译接口

使用 curl 或 Postman 测试：

```bash
curl -X POST http://localhost:8000/api/translate \
  -H "Content-Type: application/json" \
  -d '{
    "texts": ["规则", "如何取得点数", "注意事项"],
    "model": "gemini-1.5-flash"
  }'
```

## ☁️ 部署到 Vercel（推荐）

### 为什么选择 Vercel？

- ✅ 完全免费
- ✅ 自动 HTTPS
- ✅ 全球 CDN
- ✅ 自动部署（连接 GitHub）
- ✅ 简单易用

### 部署步骤

#### 方法 1：使用 Vercel CLI（推荐）

```bash
# 安装 Vercel CLI
npm install -g vercel

# 登录
vercel login

# 部署
vercel

# 设置环境变量
vercel env add GEMINI_API_KEY
# 然后粘贴您的 API Key

# 生产环境部署
vercel --prod
```

#### 方法 2：通过 Vercel 网页

1. 访问 https://vercel.com
2. 用 GitHub 账号登录
3. 点击 **"Import Project"**
4. 选择您的 GitHub 仓库
5. 配置环境变量：
   - 名称：`GEMINI_API_KEY`
   - 值：您的 API Key
6. 点击 **"Deploy"**

### 部署后

您会获得一个域名，例如：
```
https://your-project.vercel.app
```

API 地址：
```
https://your-project.vercel.app/api/translate
```

## 🔧 前端集成

部署后，需要更新前端代码，将 API 地址指向您的后端服务。

### 修改 `翻译工具.html`

找到翻译函数，将 API 调用改为：

```javascript
// 替换原来直接调用 Gemini API 的代码
async function startTranslation() {
    const inputText = document.getElementById('inputText').value.trim();
    const lines = inputText.split('\n').filter(line => line.trim());
    
    // 调用后端 API
    const response = await fetch('https://your-backend.vercel.app/api/translate', {
        method: 'POST',
        headers: {
            'Content-Type': 'application/json',
        },
        body: JSON.stringify({
            texts: lines,
            model: 'gemini-1.5-flash'
        })
    });
    
    const result = await response.json();
    
    if (result.success) {
        translationResults = result.data;
        displayResults();
    } else {
        alert('翻译失败：' + result.message);
    }
}
```

## 🔒 安全建议

### 1. API Key 保护

- ✅ 永远不要将 `.env` 文件提交到 Git
- ✅ 使用 `.gitignore` 排除敏感文件
- ✅ 在部署平台使用环境变量功能

### 2. 添加速率限制

安装 `slowapi`：

```bash
pip install slowapi
```

在 `main.py` 中添加：

```python
from slowapi import Limiter, _rate_limit_exceeded_handler
from slowapi.util import get_remote_address
from slowapi.errors import RateLimitExceeded

limiter = Limiter(key_func=get_remote_address)
app.state.limiter = limiter
app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)

@app.post("/api/translate")
@limiter.limit("10/minute")  # 每分钟最多 10 次请求
async def translate_texts(request: Request, ...):
    ...
```

### 3. 添加身份验证

可以使用 API Key 验证：

```python
from fastapi import Header, HTTPException

API_KEYS = ["your-secret-key-1", "your-secret-key-2"]

@app.post("/api/translate")
async def translate_texts(
    request: TranslationRequest,
    x_api_key: str = Header(...)
):
    if x_api_key not in API_KEYS:
        raise HTTPException(status_code=401, detail="Invalid API Key")
    ...
```

## 🐛 故障排除

### 问题 1：ModuleNotFoundError

**解决方案**：确保已安装所有依赖

```bash
pip install -r requirements.txt
```

### 问题 2：GEMINI_API_KEY not found

**解决方案**：检查 `.env` 文件是否正确配置

```bash
# 查看环境变量
python -c "from dotenv import load_dotenv; import os; load_dotenv(); print(os.getenv('GEMINI_API_KEY'))"
```

### 问题 3：CORS 错误

**解决方案**：确保 CORS 配置正确

在 `main.py` 中：

```python
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://your-frontend-domain.com"],  # 改为您的前端域名
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

### 问题 4：Vercel 部署失败

**解决方案**：

1. 检查 `vercel.json` 配置
2. 确保 `requirements.txt` 格式正确
3. 查看 Vercel 部署日志
