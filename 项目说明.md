# 🎮 游戏运营策划工具 - 项目说明

## 📖 项目概述

这是一个为游戏运营策划人员设计的工具集，包含礼包配置、掉落配置和多语言翻译功能。

## 🛠️ 工具介绍

### 1. 奖励配置工具 (礼包配置工具)

**功能**：
- 快速生成礼包奖励 JSON 配置
- 支持多个价格档位 (0.99, 1.99, ...)
- 自动计算 XP 和帮派宝箱
- 支持自定义物品添加
- 拖拽排序功能
- JSON 压缩/展开

**技术**：纯前端 HTML/CSS/JavaScript

### 2. 掉落配置工具

**功能**：
- 配置随机掉落奖励
- 设置物品权重
- 自动计算掉落概率
- 支持多物品配置
- 拖拽排序
- JSON 导出

**技术**：纯前端 HTML/CSS/JavaScript

### 3. 多语言翻译工具

**特点**：
- ✅ API Key 安全存储在服务器端
- ✅ 使用最新 Gemini 2.5 Flash 模型
- ✅ 支持批量翻译到 17 种语言
- ✅ 高质量 AI 翻译，理解上下文
- ✅ 可添加访问控制和监控
- ✅ 适合个人和团队使用

**技术栈**：
- 后端：FastAPI (Python)
- AI：Google Gemini 2.5 Flash
- 部署：本地/Vercel/Railway/Heroku

**文件**：
- 后端：`main.py`
- 前端：`翻译工具-后端版.html`

## 🚀 快速开始

### 方式 1：仅使用前端工具

**适用于**：礼包配置、掉落配置

1. 直接在浏览器中打开对应的 HTML 文件
2. 无需任何配置，即可使用

### 方式 2：使用翻译工具（需要后端服务）

**第一步：启动后端服务**

```bash
# Linux/Mac
./start.sh

# Windows
start.bat

# 或手动启动
python main.py
```

**第二步：配置前端**

打开 `翻译工具-后端版.html`，确认 API 地址为 `http://localhost:8000`

**第三步：开始使用**

输入中文 → 点击翻译 → 复制结果

## 🔧 本地开发

### 环境要求

- **前端工具**：任何现代浏览器
- **后端服务**：
  - Python 3.9+
  - pip 包管理器

### 安装依赖

```bash
# 创建虚拟环境（推荐）
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt
```

### 配置环境变量

```bash
# 复制模板
cp .env.example .env

# 编辑 .env 文件
nano .env
```

在 `.env` 中添加：

```
GEMINI_API_KEY=your_api_key_here
```

**获取 API Key**：https://aistudio.google.com/app/apikey

### 运行服务

```bash
# 启动后端
python main.py

# 或使用启动脚本
./start.sh  # Linux/Mac
start.bat   # Windows
```

### 测试 API

```bash
# 运行测试脚本
python test_api.py

# 或访问 API 文档
open http://localhost:8000/docs
```

## 📊 API 接口说明

### GET /health

健康检查

**响应**：
```json
{
  "status": "healthy",
  "api_configured": true,
  "supported_languages": 17
}
```

### POST /api/translate

翻译接口

**请求**：
```json
{
  "texts": ["规则", "如何取得点数"],
  "model": "gemini-1.5-flash"
}
```

**响应**：
```json
{
  "success": true,
  "data": [
    {
      "en": "Rules",
      "fr": "Règles",
      ...
    }
  ],
  "message": "成功翻译 2 条文本"
}
```

### GET /api/languages

获取支持的语言列表

**响应**：
```json
{
  "codes": ["en", "fr", "de", ...],
  "names": {
    "en": "English",
    ...
  },
  "count": 17
}
```

## 🐛 常见问题

### Q: 后端启动失败

**A:** 检查：
1. Python 版本是否 >= 3.9
2. 依赖包是否安装
3. `.env` 文件是否配置
4. 端口 8000 是否被占用

### Q: 翻译失败

**A:** 检查：
1. GEMINI_API_KEY 是否正确
2. 网络是否可以访问 Google 服务
3. API 配额是否已用完
4. 查看后端日志了解详细错误

### Q: CORS 错误

**A:** 
- 如果从本地文件打开 HTML（`file://`），可能有跨域限制
- 建议使用 Live Server 或部署到服务器

### Q: 如何添加新语言？

**A:** 
1. 修改 `main.py` 中的 `LANGUAGE_CODES` 和 `LANGUAGE_FULL_NAMES`
2. 修改前端 HTML 中的 `languages` 和 `displayCodes`
3. 重启服务
