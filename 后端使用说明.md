### 后端文件
- `main.py` - FastAPI 后端服务（核心 API）

- `翻译工具-后端版.html` - 调用后端 API 的前端页面

## ⚡ 5 分钟快速启动

### 步骤 1：安装 Python 依赖

```bash
pip install -r requirements.txt
```

### 步骤 2：配置 API Key

```bash
# 复制配置模板
cp .env.example .env

# 编辑 .env 文件
nano .env
# 或使用其他编辑器打开
```

在 `.env` 文件中填入：

```
GEMINI_API_KEY=AIzaSy...你的API密钥
```

**获取 API Key**：https://aistudio.google.com/app/apikey

### 步骤 3：启动后端服务

```bash
python main.py
```

看到以下信息说明启动成功：

```
✅ 游戏运营策划工具 - 翻译服务
📡 服务地址：http://localhost:8000
📖 API 文档：http://localhost:8000/docs
```

### 步骤 4：打开前端页面

用浏览器打开：`翻译工具-后端版.html`

或访问：`file:///路径/翻译工具-后端版.html`

### 步骤 5：开始翻译

1. 确认 API 地址为 `http://localhost:8000`
2. 输入中文文本
3. 点击"开始翻译"
4. 完成！

## 🌐 部署到云端（推荐 Vercel）

### 为什么要部署？

- ✅ 任何设备都能访问
- ✅ HTTPS 安全连接
- ✅ 全球 CDN 加速
- ✅ 完全免费

### Vercel 快速部署（3 分钟）

```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录
vercel login

# 3. 部署
vercel

# 4. 添加环境变量
vercel env add GEMINI_API_KEY
# 粘贴您的 API Key

# 5. 生产环境部署
vercel --prod
```

部署成功后，您会获得一个域名：

```
https://your-project.vercel.app
```

更新前端 API 地址为：`https://your-project.vercel.app`

## 🔄 两种模式对比

### 前端直接调用模式（旧版）

**文件**：`翻译工具.html`

**优点**：
- ✅ 无需后端服务器
- ✅ 部署简单

**缺点**：
- ❌ API Key 暴露在前端
- ❌ 安全性较低
- ❌ 无法统一管理

### 后端 API 模式（推荐）

**文件**：`翻译工具-后端版.html` + `main.py`

**优点**：
- ✅ API Key 安全存储在服务器
- ✅ 可以添加访问控制
- ✅ 统一管理和监控
- ✅ 可以添加缓存等优化

**缺点**：
- ❌ 需要部署后端服务

## 🐛 常见问题

### Q: 启动后端时报错 "GEMINI_API_KEY 未设置"

**A:** 请检查：

1. `.env` 文件是否存在
2. `.env` 文件中是否配置了 `GEMINI_API_KEY`
3. 文件格式是否正确（不要有多余的引号或空格）

```bash
# 正确格式
GEMINI_API_KEY=AIzaSyABCDEF123456...

# 错误格式（不要这样）
GEMINI_API_KEY="AIzaSy..."
GEMINI_API_KEY = AIzaSy...
```

### Q: 前端显示"翻译失败：Failed to fetch"

**A:** 可能原因：

1. **后端未启动**
   - 检查后端是否正在运行
   - 访问 http://localhost:8000/health 验证

2. **API 地址错误**
   - 确认输入的地址是 `http://localhost:8000`
   - 不要加 `/api/translate`（前端会自动添加）

3. **CORS 问题**
   - 如果从文件系统打开 HTML（`file://`），可能有跨域限制
   - 建议使用 Live Server 或部署到服务器

### Q: 翻译速度慢

**A:** 
- Gemini API 每行翻译需要 1-2 秒
- 10 行文本约需 10-20 秒
- 这是为了避免超过 API 限额（每分钟 60 次）
- 已经在代码中添加了 1 秒延迟

### Q: 如何修改支持的语言？

**A:** 在 `main.py` 中修改：

```python
LANGUAGE_CODES = ['en', 'fr', 'de', ...]  # 修改这里
LANGUAGE_FULL_NAMES = {
    'en': 'English',
    ...  # 修改这里
}
```

同时修改前端 `翻译工具-后端版.html` 中的 `languages` 和 `displayCodes`。

## 📈 后续优化建议

### 1. 添加缓存

避免重复翻译相同的文本：

```bash
pip install redis
```

```python
# 在 main.py 中添加 Redis 缓存
import redis

cache = redis.Redis(host='localhost', port=6379, db=0)
```

### 2. 添加速率限制

防止滥用：

```bash
pip install slowapi
```

### 3. 添加身份验证

使用 API Key 验证：

```python
from fastapi import Header

@app.post("/api/translate")
async def translate_texts(
    request: TranslationRequest,
    x_api_key: str = Header(...)
):
    if x_api_key != "your-secret-key":
        raise HTTPException(401, "Unauthorized")
    ...
```

### 4. 添加日志和监控

```bash
pip install loguru
```
